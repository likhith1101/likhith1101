## About Me

I build hardware — from RTL to silicon and from FPGA to inference.

My work sits at the intersection of **digital design**, **physical design**, and **hardware-accelerated ML**. I've taken a processor from Verilog source all the way to a DRC-clean GDSII layout using open-source EDA tools, implemented a complete RISC-V pipeline with hazard handling, and deployed a CNN accelerator on an FPGA that matches ResNet-50 accuracy at a fraction of the resource cost.

## Projects

### [RISC-V Pipelined Processor](https://github.com/likhith1101/riscv-processor-design)
> 5-stage RV32I pipeline with full hazard resolution — data forwarding, load-use stall detection, branch flush — extended with a custom modulo multiplier hardware unit

[![Pipeline](https://img.shields.io/badge/Pipeline-5--Stage-00C8D7?style=flat-square)]()
[![Forwarding](https://img.shields.io/badge/Hazards-Forwarding%20%2B%20Stall-00D68F?style=flat-square)]()
[![Vivado](https://img.shields.io/badge/Simulator-Vivado%20XSim-FFB830?style=flat-square)]()

- **IF → ID → EX → MEM → WB** pipeline with pipeline registers
- **Forwarding unit** — resolves RAW hazards with 2-bit `forward_rs1/rs2` control
- **Stall unit** — detects load-use hazards, inserts bubble, freezes PC
- **Branch unit** — resolves branches at end of EX, flushes on taken
- **Modulo multiplier** — Barrett-reduction digit-serial hardware unit (`N=12, m=4`)

> Custom CNN compiled through Tensil into a systolic-array accelerator on Zynq-7020 — matching ResNet-50 accuracy at 31% BRAM and ~3× faster than CPU

[![PYNQ-Z2](https://img.shields.io/badge/Platform-Zynq--7020-00C8D7?style=flat-square)]()
